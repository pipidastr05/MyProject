# Стандарты HTML

## Зачем нам это?
1. Читаемость - другой человек должен быстро понять вашу структуру
2. Поддержка - легче вносить изменения через полгода
3. Доступность (accessibility,a11y ) - правильная семантика - это не прихоть
4. Базовый профессионализм - так делают в индустрии

## Актуальность HTML-тегов и атрибутов
1. Can i use? - сервис для проверки поддержки отдельных элементов в разных браузерах:
   https://caniuse.com/
2. Canincludе - сервис для проверки вложенности тегов:
   https://caninclude.onrender.com/
3. W3C - можно проверить правильность всей разметки, прикрепив html-документ, вставив код или ссылку на страницу в интернете. **Лучше всегда через него проверяться**: https://validator.w3.org/

## 1) Cемантика
**Семантика** - использование HTML-тегов по их назначению и показательная характеристика качества веб-приложения. Такой сайт будет лучше продвигаться поисковыми системами, будет удобнее для пользователей скрин-ридеров.
- `<header>` - для выделения верхней части страницы, секции или любого другого блока. Также используется для заголовках на странице и в карточках товаров.
- `<footer>` - нижняя часть страницы, подвал; а также для любой нижней части секции или любого другого блока.
- `<main>` - может быть только **один** на странице, между шапкой и подвалом.
- `<section>` - независимые функциональные блоки. **Обязан** внутри себя **содержать заголовок** (обычно H2). Разрешается вкладывать друг в друга.
- `<article>` - для выделения самодостаточного блока какой-то конкретной сущности: карточка товара, карточка новости, главный баннер, виджет и подобное.  
  Лайфхак: если в теории блок можно вынести на другой сайт в качестве рекламного баннера, то это `<article>`
- `<aside>` - косвенное содержимое, вспомогательная инфа (договорились, что у нас - всегда боковушка)
- `<nav>` - блоки навигации - обычно для основного меню сайта + "хлебные крошки". Как правило, должен быть хотя бы один на сайте.

`<div>` и `<span>` - только при необходимости (обычно для применения классов) - они, в отличие от вышеперечисленных, не имеют смысловой нагрузки для поисковиков

## 2) Заголовки
- H1 - может быть только один на странице
- Все заголовки идут строго по порядку, без внезапных попусков
- Если смотреть только на заголовки страницы, то должно получиться “оглавление”

## 3) Список
- любая последовательность однотипных элементов (список пунктов меню в шапке сайта, последовательность кнопок)

## 4) Форматирование текста
- договариваемся не применять на уровне HTML, делаем силами CSS

## 5) ссылки и кнопки 
- кнопки используются только для обработки пользовательских действий на сайте (обычно в рамках одной страницы); все остальное - через ссылки
- обязательно используем правильные ссылки, выделяют 5 видов: якорные, скачивания, эллектронной почты, номер телефона, переход на другую страницу (target="_blank" для открытия в новом окне).

## 6) Изображения:
- обязательно использовать alt (альтернативное описание изображения). Заполняем, только если картинка несет пользователю ценность. Не добавлять alt - нельзя!
- обязательно использование атрибутов width и height - для правильной загрузки страницы
- обязательно loading=”lazy” - для оптимальной загрузки изображений в браузере.

## 7) Медиаконтент
- для видео, как и для img - обязательно использование атрибутов width и height для правильной загрузки страницы
- к аудио - требований нет
- `<iframe>` - тоже обязательно width и height

## 8) Формы
- используем правильные значения для атрибута `type=""` - отвечает за настройку вариаций тега `<input>`
- используем правильные значения для атрибута `inputmode=""` - указывает альтернативную клавиатуру браузерам на мобильных устройствах
- атрибуты name и id стоит заполнять, ориентируясь на список зарезервированных браузеров имен полей - это влияет на автозаполнение (autocomplete).  
  Возможные значения: https://doka.guide/html/autocomplete/

## 9) Векторная графика
- используем для иконок, логотипов, дизайнерских украшательств в виде деталек, фигур, линий.
- лучше вставлять код `<svg>`, а не как изображения - это дает большую гибкость над визуальной составляющей (цвет, анимация и тд)

## 10) Глобальные атрибуты
- `class=""` - общие правила по неймингу классов: https://github.com/aleksanderlamkov/html-classnames
- `id=""` - должен быть уникальным в рамках одной страницы. Значение - слово или набор символов с минимум 1 буквой. Не может начинаться с цифры и содержать пробел.
- `title=""` - стоит добавлять для всех интерактивных кнопок - добавляет браузерную подсказку (тултип) при наведении
